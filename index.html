    Introduction

    Welcome to the Unsolved API. (Version 1)

    Keys

    • api-key
    • access_token
    • refresh_token

    Header

    • "Header: x-api-key: api-key" is a required header to access all apis

    Registration

    Signup

    This endpoint is to register user. Generates otp and sends otp automatically

    URL: "localhost:3000/api/v1/registrations/"
METHOD: POST

    The request data JSON structure:

    {
   "user" : {
        "first_name": "abcd",
        "last_name": "xyz",
        "email": "abcd@gmail.com",
        "country_code": "+91",
        "phone": "9988776655",
        "password": "abcd",
        "terms_of_service": true
   },
   "device_token": "pqrst"
}

    The above command returns JSON structured like this:

    {
    "access_token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE2Mjg3NjEzNTR9.uk_g3M_W8fV-Ov--plaLGUDeI8LDPaT-iwdmJnbX4GE",
    "refresh_token": "eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX3Rva2VuIjoieDZRNnZiNkJuNG80U2lCTGgxajNWUlJOIiwiZXhwIjoxNjYwMjk3Mjk0fQ.Hc_oMp0LvQ-BuUGi4SFyT9eNy59WdftdKOooTXGjxJw"
}

    HTTP Request

    POST localhost:3000/api/v1/registrations/

    Data Parameters

    Parameter Required Value Description
    first_name true String First name of user
    last_name true String Last name of user
    country_code true String Phone country code
    phone true String Phone number of the user
    email true String Email of user
    password true String Password of the user
    terms_of_service true boolean required true for registration
    device_token true String Device token of the mobile. Won't raise error if missing
    user parameters should be structured under key 'user'

    Verify Otp

    This endpoint is to verify otp.

    URL: "localhost:3000/api/v1/registrations/verify_otp"
METHOD: POST
HEADER: Authorization: Bearer access_token

    The request data JSON structure:

    {
    "otp": "1234"
}

    The above command returns JSON structured like this:

    {

    "msg": "otp verified"

    Authorization: Bearer access_token
